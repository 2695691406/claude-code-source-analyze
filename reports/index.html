8 大运行模式

灵活适配不同使用场景的多模式架构

1 CLI 交互模式 REPL

默认交互式终端，支持多轮对话、工具调用和实时流式输出。

2 非交互 Print 模式 --print

单次查询输出，适用于管道操作和脚本集成场景。

3 SDK 编程模式 API

程序化调用接口，通过 Node.js/Bun SDK 实现应用内集成。

4 Coordinator 多Agent模式 Multi

多智能体协作，Coordinator 编排多个子 Agent 并行处理任务。

5 Bridge 远程控制模式 Remote

Web 界面远程操控，通过 WebSocket 桥接本地 Agent。

6 Daemon 后台模式 BG

后台持续运行，守护进程模式支持长期监控和自动化任务。

7 KAIROS 助手模式 Cron

定时任务调度，基于 KAIROS 框架实现周期性自动化执行。

8 Voice 语音模式 Voice

语音交互，支持语音输入识别和语音合成输出。
