🛡️ 七层安全架构

纵深防御策略，每一层独立生效确保安全

1 工具池过滤 filterToolsByDenyRules

编译时移除危险工具，从源头控制可用工具集，不合规工具永远不会出现在候选列表中

2 输入验证 Zod Schema

基于 Zod Schema 的类型安全参数校验，确保工具接收的参数格式和类型完全正确

3 权限模式快速路径 5 种模式

manual / ask / auto / bypass / plan — 5 种权限模式快速判断，避免重复确认

4 工具级检查 Per-Tool Logic

每个工具自定义权限逻辑 — 如文件写入工具检查路径、Bash 工具检查命令安全性

5 规则匹配 7 级优先级

policy > user > project > local > flag > command > session — 7 级优先级规则匹配系统

6 PreToolUse Hooks 外部扩展

外部安全检查扩展点，允许第三方插件或企业策略在工具执行前进行拦截和审计

7 分类器 / 用户确认 最终决策

YOLO 模式自动分类放行，或交互式弹出确认对话框由用户手动决定是否执行

📦 五层压缩流水线

多策略协同，平衡信息保留与 Token 效率

1
工具结果预算
Tool Result Budget
单次工具输出超限截断 — 当某个工具返回过长结果时，按预设 Token 预算截断，保留头部和尾部关键信息
2
Snip 压缩
History Snip
旧对话历史渐进式修剪 — 从最早的消息开始逐步移除，保留最近的上下文以维持对话连贯性
3
微压缩
Microcompact
按工具类型的时间窗口清理 — 对不同类型的工具输出设置不同的保留时间窗，过期内容自动移除
4
上下文折叠
Context Collapse
读时投影，延迟物化 — 不立即删除内容，而是在读取时才决定哪些内容需要展示，实现无损压缩效果
5
自动压缩
Auto Compaction
Token 阈值触发全局压缩 — 当总上下文长度接近模型限制时，使用 LLM 对全部历史进行摘要式压缩

⚡ 性能优化策略

多维度性能优化，兼顾启动速度与运行效率

🏎️
快速路径
Fast Paths

cli.tsx 中的 --version 等命令直接输出退出，无需加载完整初始化链路，毫秒级响应

⚡
并行初始化
Parallel Init

MDM + Keychain 并行加载，Remote settings 异步获取，多个独立初始化任务同时进行

💾
Prompt 缓存稳定性
Cache Stability

工具池排序策略确保缓存键稳定，相同的工具集合始终产生相同的缓存 key，最大化缓存命中率

🔮
投机执行
Speculation

用户输入时预测并预启动 API 调用，在用户仍在输入时就开始准备请求，减少感知延迟

💤
延迟加载
Lazy Loading

shouldDefer 标记工具延迟注册，非立即需要的工具推迟到实际使用时才加载初始化

🚪
Feature Gate
Dead Code Elimination

编译时死代码消除，通过 Feature Gate 在构建阶段剔除未启用的特性代码，减小包体积

💡 设计评析

架构设计的巧妙之处与潜在改进空间

✅ 巧妙之处
  • 🚀 三阶段启动：渐进式初始化，快速路径直接退出，避免不必要的初始化开销，实现毫秒级响应
  • 🛡️ 分层安全：七层纵深防御架构，每层独立生效，即使某一层被绕过，后续层仍能提供保护
  • 📦 压缩流水线：五层策略协同工作，从精细到粗粒度逐层压缩，在信息保留与 Token 效率间取得平衡
  • 🔒 不可变状态：Zustand-like 不可变更新模式，有效避免多 Agent 场景下的竞态条件和状态不一致问题
⚠️ 潜在问题
  • 📄 main.tsx 4683 行：单文件代码量过大，包含过多职责，增加了理解、维护和代码审查的困难
  • 🏗️ 8 层架构复杂度：深层架构增加了理解和调试的复杂度，新开发者上手成本较高
  • 📉 压缩信息丢失：多层压缩策略可能丢失重要的上下文信息，影响长对话中模型的回答质量
  • 🔧 多供应商适配：API 客户端需要适配多个供应商差异，增加了维护成本和潜在的兼容性风险
