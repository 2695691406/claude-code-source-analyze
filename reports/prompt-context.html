💡 核心洞察：System Prompt as Policy
所有行为控制通过提示词注入实现，零代码路径修改。这意味着新功能、新约束、新角色都可以通过修改提示词配置来实现，无需修改任何代码逻辑。提示词的组装顺序决定了优先级：核心指令 → 工具描述 → 记忆 → Agent → 协调 → 动态上下文。

🔥 Fire-and-Forget 转录优化
转录文件写入采用非阻塞模式（fire-and-forget），不等待磁盘 I/O 完成即可继续流式处理。这确保了转录记录不会成为管道的性能瓶颈，同时保证了完整的会话记录。

🗜️ 五层上下文压缩策略

渐进式压缩管道，从工具结果到完整上下文的多级压缩

1 工具结果预算 Tool Result Budget
触发: 单次工具输出超限 策略: 截断为预算内大小 保留: 头尾信息优先，中间内容省略
2 历史修剪 History Snip
触发: 旧对话累积过多 策略: 渐进式修剪旧消息 保留: 最近 N 轮对话完整保留
3 微压缩 Microcompact
触发: 按工具类型的时间窗口 策略: 工具结果按类型批量清理 保留: 最近使用的工具结果
4 上下文折叠 Context Collapse
触发: Token 接近阈值 策略: 读时投影，延迟物化 保留: 摘要替代原文
5 自动压缩 Auto Compaction
触发: Token 超过阈值（如 80%） 策略: 分叉子 Agent 生成摘要，替换历史 保留: 完整摘要 + 最近对话
🎯 Cache-aware Compression
压缩策略感知 Prompt 缓存机制，在执行压缩操作时避免破坏已缓存的前缀内容。这确保了压缩后的对话仍然能命中 API 的 Prompt 缓存，避免因压缩导致的额外 Token 成本。

💎 Prompt缓存优化策略

通过稳定化技术最大化 Prompt 缓存命中率，降低 API 成本

🔧 工具池排序稳定化
确保工具描述的顺序在每次请求中保持一致，避免因顺序变化导致缓存键不同。工具池在初始化时排序，后续请求复用相同顺序，确保系统提示的前缀部分始终一致。
📌 系统提示稳定性
最小化动态内容在系统提示中的位置变化。静态内容（核心指令、工具描述）放在前部，动态内容（环境状态、会话信息）放在后部，确保前缀的缓存可复用性。
🗜️ 压缩缓存感知
上下文压缩时保留前缀不变，确保压缩操作不会破坏已有的缓存命中。压缩算法从后往前删除内容，保持与缓存兼容的前缀完整性。
🔀 Fork模式优化
子 Agent 继承父 Agent 的上下文前缀，复用父级的缓存。这在多 Agent 协作场景中显著减少重复的缓存创建成本，每个子 Agent 只需要缓存增量部分。

🎯 延迟优化效果
通过投机执行，用户提交消息到 API 请求发出的延迟从「上下文组装时间 + 网络延迟」降低为仅「网络延迟」。在复杂项目中，上下文组装可能涉及文件读取、CLAUDE.md 解析、工具列表构建等耗时操作，投机执行将这些操作与用户输入并行化。

❓ 为什么自定义而非使用 Redux/Zustand？
Claude Code 运行在终端环境（Ink + React），不是浏览器 DOM。标准状态管理库假设浏览器环境，而终端 UI 有不同的渲染模型和生命周期。自定义实现可以：① 避免不必要的依赖；② 适配 Ink 的渲染模型；③ 支持 CLI 特有的状态需求（如进程状态、工具执行状态）；④ 保持与 AsyncGenerator 管道的紧密集成。

⚖️ 设计评析

Prompt 工程与上下文管理的巧妙之处与潜在问题

✅ 巧妙之处
  • 🔄 AsyncGenerator 管道：实现零依赖流式处理，天然支持背压控制和取消传播
  • 🗜️ 五层压缩策略：从工具结果到完整历史的渐进压缩，平衡信息保留与 Token 效率
  • 📝 System Prompt as Policy：配置即行为，所有控制通过提示词实现，极大降低了修改成本
  • 💎 Prompt 缓存稳定性：通过工具排序和前缀稳定化，有效减少 API 缓存未命中
  • 🛡️ Zod 双重验证：编译时 + 运行时双保险，确保工具输出的完整类型安全
  • ⚡ 投机执行：预测用户行为并行准备上下文，显著提升交互响应速度
⚠️ 潜在问题
  • 🗜️ 压缩信息丢失：多层压缩可能丢失重要上下文信息，影响长对话中模型的回答质量
  • 📊 Token 估算精度：本地估算在不同模型间存在精度差异，可能导致压缩过早或过晚触发
  • 📏 系统提示过长：43+ 工具描述 + 多层配置使系统提示体积庞大，影响推理效率和成本
  • 🔀 CLAUDE.md 合并冲突：三级记忆文件的合并可能产生语义冲突，缺乏显式的冲突检测机制
  • 📈 动态上下文增长：运行时注入的动态上下文（文件快照、环境状态）增长难以精确控制
