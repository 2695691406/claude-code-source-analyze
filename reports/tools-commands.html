🧩 工具抽象层设计

Claude Code 定义了一套统一的工具接口（Tool Interface），所有工具——无论是内置工具、MCP外部工具还是自定义技能——都遵循同一套协议，实现了高度可扩展的工具生态。

统一工具接口（Unified Tool Interface）

interface Tool {
  // 身份
  name: string;
  description: string;

  // Schema — 编译时TypeScript + 运行时Zod验证
  inputSchema: ZodSchema;
  outputSchema: ZodSchema;

  // 行为谓词
  isReadOnly(): boolean;  // 默认 false (Fail-Closed)
  isConcurrencySafe(): boolean;  // 默认 false (Fail-Closed)

  // 执行
  call(input): Promise<ToolResult>;

  // 权限
  userFacingPermissions(): Permission[];
  canUse(): boolean;

  // 渲染
  renderToolUseMessage(): ReactNode;
  renderResultMessage(): ReactNode;
}

默认安全策略：Fail-Closed

所有工具默认被视为非并发安全、非只读。只有显式声明安全的工具才能并发执行或跳过权限检查。

isReadOnly() → false isConcurrencySafe() → false 显式声明 → 安全

🛡️ 权限决策流水线

每次工具调用都要经过 7 步权限决策流水线，从工具池过滤到最终用户确认，形成纵深防御体系。

1
工具池过滤
filterToolsByDenyRules 移除受限工具，确保被禁用的工具不会出现在候选列表中
2
输入验证
Zod Schema 验证工具参数，拒绝不合法的输入数据
3
权限模式快速路径
manual — 总是询问 ask — 记忆选择 auto — YOLO模式 bypass — 跳过 plan — 仅规划
4
工具级权限检查
每个工具自定义 canUse() 逻辑，如工具特定的前置条件检查
5
规则匹配
policy > user > project > local > flag > command > session
7级优先级规则链，高优先级规则覆盖低优先级
6
PreToolUse Hooks
外部安全扩展点，允许第三方安全工具在工具执行前进行审查
7
分类器 / 用户确认
Bash YOLO 分类器（AI 判断命令安全性）或交互式 UI 确认

🚪 Feature Gate 机制

Claude Code 使用编译时 Feature Gate 进行条件编译，在构建阶段消除未启用功能的代码，与传统运行时 Feature Flag 形成鲜明对比。

编译时消除 vs 运行时Feature Flag

特性编译时 Feature Gate运行时 Feature Flag
代码消除✅ 死代码完全移除❌ 代码保留在包中
字符串泄露✅ 不泄露❌ 可能泄露
包体积✅ 显著减小❌ 无变化
动态切换❌ 需重新构建✅ 运行时切换
实现方式Bun构建时评估条件运行时条件判断

20+ 特性标识

条件编译控制的功能模块，在构建阶段由 Bun 打包工具评估，未启用的功能代码被完全移除：

💡 设计评析

✅ 巧妙之处

  • 统一的工具抽象接口简化扩展——新增工具只需实现统一接口
  • Fail-Closed 默认安全策略——工具默认不可信
  • 7层权限纵深防御——多层安全保障
  • Bash 23+ 模式检测保障安全——两阶段检测架构
  • MCP协议实现真正的工具生态开放——5种传输协议支持
  • Feature Gate 编译时优化减小包体积——死代码完全消除

⚠️ 潜在问题

  • 43+ 工具的维护成本高——每个工具需独立维护
  • MCP动态工具的安全边界模糊——外部工具信任问题
  • 权限规则7级优先级理解成本高——调试困难
  • YOLO分类器可能存在误判——安全性取决于AI判断质量
  • 命令数量庞大(86+)增加学习曲线——用户需记忆大量命令
